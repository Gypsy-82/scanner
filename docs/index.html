What it scans

Six modules.
One dashboard.

Every module runs server-side inside a read-only Docker container. Nothing touches a third-party service unless you opt in with your own API key.

File Scanner
Upload a PDF or document. ClamAV antivirus + pdfid structural analysis detect malware, embedded scripts, and obfuscated payloads.
URL Scanner
Check a URL against URLhaus, VirusTotal, and Google Safe Browsing. Surfaces known malware distribution, phishing, and C2 infrastructure.
Site Audit
Full security posture check: response headers, SSL/TLS config, open directories, sensitive path exposure, CMS detection, WordPress-specific hardening checks.
OSINT
DNS records, subdomain enumeration, reverse IP lookup, certificate transparency log search. Maps the external attack surface of any domain.
Crawl
Spider a site to discover hidden paths, sensitive probes, and broken links. Surfaces resources the site owner may not know are publicly accessible.
Inspector
Burp Suite-style response analysis: response headers, hardcoded credentials, JS secrets, DOM XSS sinks, mixed content, SRI violations, and 13 more checks per request.

Zero-trust by design

Your credentials never
leave your server.

No SaaS account. No API token registered with us. No telemetry. Here is exactly what happens when you set it up.

  1. 1
    You run php setup.php on your server. It asks for your email and password interactively in your terminal.
  2. 2
    It generates a random pepper, HMAC-SHA256-hashes your email, and bcrypt-hashes your password at cost-13 (~1 second per attempt).
  3. 3
    It writes config.php with the hashed values only. No plain-text credentials are stored anywhere, ever.
  4. 4
    config.php lives on your VPS at /opt/scanner/ — above the web root, outside the Docker image, never in this repo.
  5. 5
    If an attacker gets the hash without the pepper, hashcat is useless. Both values are required to crack it.
  6. 6
    Every login generates 6 new random URL tokens for scan modules. The paths change every session — nothing to enumerate.

Defense in depth

Four isolation layers.

Even if one layer is bypassed, the next one holds. The scanner is treated as untrusted by the host at every level.

Network

  • Loopback-only binding
  • iptables subnet isolation
  • Public DNS only (8.8.8.8)
  • Cloud metadata API blocked
  • All private ranges blocked

Container

  • Read-only root filesystem
  • tmpfs for /tmp and sessions
  • ALL capabilities dropped
  • no-new-privileges flag
  • Resource limits enforced

PHP Runtime

  • open_basedir restricted
  • shell_exec, system disabled
  • phpinfo() disabled
  • Sessions in RAM only
  • Hardened php.ini

Authentication

  • HMAC-SHA256 + bcrypt
  • Random session route tokens
  • IP-based rate limiting
  • Inactivity auto-logout
  • No predictable URL paths

Get running

Install in minutes.

Needs Docker, nginx, and PHP CLI on an Ubuntu / Debian VPS. SSL via Certbot. Or run the automated script and answer two prompts.

1

Clone and build

git clone https://github.com/Gypsy-82/scanner.git
cd scanner
docker compose build
2

Create the config directory

This directory lives above the web root and is never served publicly.

sudo mkdir -p /opt/scanner/logs
sudo chmod 750 /opt/scanner
3

Set your credentials

Type your email and password directly into your terminal. They are never stored in plain text.

sudo php setup.php
# Enter email, password (16+ chars), optional API keys

sudo mv config.php /opt/scanner/config.php
sudo chmod 640 /opt/scanner/config.php
sudo rm setup.php
4

Start the container and apply network isolation

docker compose up -d
sudo bash docker/isolate.sh
5

Configure nginx and get an SSL certificate

sudo cp docker/nginx-proxy.conf /etc/nginx/sites-available/scanner.conf
# Edit the file and replace scanner.yourdomain.com with your domain
sudo ln -s /etc/nginx/sites-available/scanner.conf /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d your.domain.com

Or run everything in one command:

sudo bash install.sh your.domain.com

You will still be prompted to enter your email and password interactively.
